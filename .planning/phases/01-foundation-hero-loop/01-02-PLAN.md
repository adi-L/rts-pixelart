```typescript
import Phaser from 'phaser';
import {
  HERO_SPEED, HERO_SCALE, HERO_BODY_WIDTH, HERO_BODY_HEIGHT,
  HERO_IDLE_FRAMES_START, HERO_IDLE_FRAMES_END, HERO_IDLE_FPS,
  HERO_WALK_FRAMES_START, HERO_WALK_FRAMES_END, HERO_WALK_FPS,
  SPRITE_HERO
} from '../constants';
```

The class should:

1. **Constructor** `constructor(scene: Phaser.Scene, x: number, y: number)`:
   - Call `scene.physics.add.sprite(x, y, SPRITE_HERO)` to create the sprite
   - Store the sprite as `this.sprite: Phaser.Physics.Arcade.Sprite`
   - Set scale to `HERO_SCALE` (2x)
   - Set physics body size to `HERO_BODY_WIDTH` x `HERO_BODY_HEIGHT` (24x48) centered on sprite
   - Set `this.sprite.setCollideWorldBounds(true)` per INFRA-03
   - Set `(this.sprite.body as Phaser.Physics.Arcade.Body).setAllowGravity(false)` -- no jumping per D-05
   - Create cursor keys: `this.cursors = scene.input.keyboard!.createCursorKeys()`
   - Create WASD keys: `this.wasd = scene.input.keyboard!.addKeys({ up: 'W', down: 'S', left: 'A', right: 'D' }) as { left: Phaser.Input.Keyboard.Key; right: Phaser.Input.Keyboard.Key; up: Phaser.Input.Keyboard.Key; down: Phaser.Input.Keyboard.Key }`

2. **createAnimations()** -- call once after construction:
   - Create `hero-idle` animation: `scene.anims.create({ key: 'hero-idle', frames: scene.anims.generateFrameNumbers(SPRITE_HERO, { start: HERO_IDLE_FRAMES_START, end: HERO_IDLE_FRAMES_END }), frameRate: HERO_IDLE_FPS, repeat: -1 })`
   - Create `hero-walk` animation: `scene.anims.create({ key: 'hero-walk', frames: scene.anims.generateFrameNumbers(SPRITE_HERO, { start: HERO_WALK_FRAMES_START, end: HERO_WALK_FRAMES_END }), frameRate: HERO_WALK_FPS, repeat: -1 })`
   - Note: Frame ranges are initial guesses from spritesheet inspection. The executor MUST visually check the spritesheet and adjust if animations look wrong.

3. **update()** -- called each frame from Game scene:
   - Check `this.cursors.left.isDown || this.wasd.left.isDown` -> `this.sprite.setVelocityX(-HERO_SPEED)`, `this.sprite.setFlipX(true)`, `this.sprite.play('hero-walk', true)`
   - Check `this.cursors.right.isDown || this.wasd.right.isDown` -> `this.sprite.setVelocityX(HERO_SPEED)`, `this.sprite.setFlipX(false)`, `this.sprite.play('hero-walk', true)`
   - Else -> `this.sprite.setVelocityX(0)`, `this.sprite.play('hero-idle', true)`

4. **Expose** `this.sprite` as a public property so Game.ts can pass it to camera.startFollow and physics.add.overlap.

5. **Export** the class as a named export: `export class Hero { ... }`

} from '../constants';

**Class properties (typed at class level):**
```typescript
export class Game extends Scene {
  private hero!: Hero;
  private skyLayer!: Phaser.GameObjects.TileSprite;
  private midLayer!: Phaser.GameObjects.TileSprite;
  private ground!: Phaser.GameObjects.Rectangle;
  private groundEdge!: Phaser.GameObjects.Rectangle;
```

**constructor:** `super('Game');`

**create() method:**

1. **Physics world bounds** (INFRA-03 -- MUST be set BEFORE creating any physics bodies):
   `this.physics.world.setBounds(0, 0, WORLD_WIDTH, WORLD_HEIGHT);`

2. **Sky layer** (depth 0, per D-03 and UI-SPEC Parallax Layers):
   - Create a canvas-sized TileSprite: `this.skyLayer = this.add.tileSprite(0, 0, this.scale.width, this.scale.height, 'background').setOrigin(0, 0).setScrollFactor(0).setDepth(0).setTint(COLOR_SKY);`
   - CRITICAL: TileSprite MUST be canvas-sized (this.scale.width x this.scale.height), NOT world-sized. World-sized TileSprites cause GPU memory bloat (Pitfall 1 in RESEARCH.md).
   - If the 'background' texture does not look right as a sky, create a Rectangle instead: `this.add.rectangle(0, 0, this.scale.width, this.scale.height, COLOR_SKY).setOrigin(0, 0).setScrollFactor(0).setDepth(0);`

3. **Mid-ground layer** (depth 1, scroll factor 0.3 per PARALLAX_MID):
   - `this.midLayer = this.add.tileSprite(0, 0, this.scale.width, this.scale.height, 'background').setOrigin(0, 0).setScrollFactor(0).setDepth(1).setTint(COLOR_MID);`
   - Set scrollFactor(0) because we manually update tilePositionX in update() for parallax

4. **Ground** (depth 2, full world width per D-01):
   - Ground rectangle: `this.ground = this.add.rectangle(WORLD_WIDTH / 2, GROUND_Y + GROUND_HEIGHT / 2, WORLD_WIDTH, GROUND_HEIGHT, GROUND_COLOR).setDepth(2);`
   - Ground top edge (2px highlight for depth per UI-SPEC): `this.groundEdge = this.add.rectangle(WORLD_WIDTH / 2, GROUND_Y, WORLD_WIDTH, 2, GROUND_EDGE_COLOR).setDepth(2);`
   - Note: Ground is a visual element. No physics body on ground -- hero has gravity disabled (D-05), so no ground collision needed.

5. **Hero** (depth 3):
   - `this.hero = new Hero(this, HERO_START_X, GROUND_Y - HERO_BODY_HEIGHT / 2);`
   - Import HERO_BODY_HEIGHT from constants for the Y calculation (hero bottom aligns with ground top)
   - `this.hero.createAnimations();`
   - `this.hero.sprite.setDepth(3);`

6. **Camera** (per HERO-02, UI-SPEC Camera section):
   - `this.cameras.main.setBounds(0, 0, WORLD_WIDTH, WORLD_HEIGHT);`
   - `this.cameras.main.startFollow(this.hero.sprite, true, CAMERA_LERP, CAMERA_LERP);`
   - `this.cameras.main.setDeadzone(CAMERA_DEADZONE_WIDTH, 0);`

7. **Clean shutdown** (INFRA-04):
   ```typescript
   this.events.once('shutdown', () => {
     EventBus.removeAllListeners();
   });
   ```

**update() method:**
1. Call `this.hero.update();` to process keyboard input
2. Update mid-ground parallax: `this.midLayer.tilePositionX = this.cameras.main.scrollX * PARALLAX_MID;`
